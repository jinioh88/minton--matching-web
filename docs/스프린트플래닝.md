# 스프린트 플래닝 (Sprint Planning)

> 참조: `docs/제품백로그.md`, `docs/요구사항분석.md`

## 스프린트 개요

- **스프린트 주기**: 2주
- **리뷰**: 각 스프린트 종료 시 사용자 리뷰 후 다음 스프린트 진행

---

## Sprint 1: 인프라 및 사용자 기초 (2주)

### 목표
프로젝트 기초를 완성하고, OAuth2 소셜 로그인·프로필 조회가 동작하는 상태를 만든다.

### 프론트엔드
| 작업 | 상세 | 백로그 ID |
|------|------|------------|
| 프로젝트 셋업 | React/Next 초기화 | - |
| 로그인 화면 | OAuth2 소셜 로그인 버튼(카카오/네이버/구글/애플) | B2-1, B2-2 |
| 신규 가입 프로필 | OAuth2 최초 로그인 시 닉네임, 관심 지역 입력 | B2-6 |
| 프로필 화면 | 내 프로필 조회/수정 | B2-4, B2-6 |
| 타인 프로필 화면 | 다른 사용자 프로필 조회 | B2-5 |

### 완료 조건
- [ ] OAuth2 로그인 후 JWT 발급 및 프로필 조회/수정 가능
- [ ] 신규 사용자 프로필 입력 후 서비스 이용 가능
- [ ] 다른 사용자 프로필 조회 가능

---

## Sprint 2: 매칭 핵심 및 파일 업로드 (2주)

### 목표
매칭 생성·목록·상세가 동작하고, AWS S3를 이용한 이미지 업로드(프로필, 매칭)가 가능하다.

### 프론트엔드
| 작업 | 상세 | 백로그 ID |
|------|------|------------|
| 프로필 이미지 업로드 | 프로필 수정 시 이미지 선택 → 업로드 | B8-2 |
| 매칭 만들기 화면 | 폼(제목, 장소, 일시, 인원, 급수, 코트비, **이미지**) | B3-1, B3-2, B8-3 |
| 매칭 목록 화면 | 카드 리스트, 필터, 관심 지역 전환 | B3-3, B3-4 |
| 매칭 상세 화면 | 상세 정보, 방장 프로필, 참여자 현황 | B3-5, B3-6 |
| 네비게이션 | 홈, 매칭 만들기, 프로필 탭 | - |

### 완료 조건
- [ ] 프로필 이미지 Multipart 업로드 → S3 저장 → URL 반영
- [ ] 매칭 생성 시 이미지 업로드 가능
- [ ] 매칭 생성 후 목록에 노출
- [ ] 필터로 목록 조회 가능
- [ ] 매칭 상세에서 모든 정보 확인 가능

---

### 목표
참여 신청, 수락/거절, 대기열, 취소가 end-to-end로 동작한다.

### 프론트엔드

|------|------|------------|
| 참여 신청 UI | 참여/대기 버튼, 멘트 입력 | B4-1, B4-2 |
| 방장 신청 관리 | 신청 목록, 수락/거절 버튼 | B4-3 |
| 참여 취소 | 내 참여 취소 | B4-4 |
| 상태 표시 | 수락 대기 중, 확정, 대기열 등 | B4-1~5 |

### 완료 조건
- [ ] 참여 신청 → 방장 수락 → 확정 흐름 동작
- [ ] 정원 초과 시 대기 신청 가능
- [ ] 취소 시 대기열 1번 승인 기회 부여
