---

### 2단계: API 클라이언트 및 인증 기초 (Foundation)

**지시 내용:**

> "백엔드와 JWT로 통신할 예정입니다. **6번(API 클라이언트 설정)**의 Axios Interceptor 구조를 미리 구현해 주시고, Zustand로 유저 정보를 담을 스토어를 만들어 주세요."

**통신 구조:**

```
[프론트엔드]                    [백엔드]
   Next.js          ←──JWT──→   Spring Boot
   - Axios Client      HTTP      - REST API
   - Zustand (유저)    /api/*    - JWT 발급·검증
   - Interceptor
```

**산출물:**

- `src/lib/api.ts` — Axios 인스턴스 + Request/Response Interceptor
- `src/stores/authStore.ts` (또는 `userStore.ts`) — Zustand 유저 스토어

## 6. API 클라이언트 설정

Step 4.1에서 Axios를 설치했으므로, Axios 인스턴스 기반으로 구성합니다. **Axios Interceptor**를 사용하면 모든 요청에 `Authorization` 헤더를 자동으로 붙일 수 있어 편리합니다.

### 6.1 Axios 인스턴스 및 Interceptor

`src/lib/api.ts` 예시:

```typescript
import axios, { AxiosError } from 'axios';

const API_BASE = process.env.NEXT_PUBLIC_API_BASE_URL || 'http://localhost:8080/api';

export const apiClient = axios.create({
  baseURL: API_BASE,
  headers: { 'Content-Type': 'application/json' },
  withCredentials: true, // 쿠키 기반 인증 시
});

// Request Interceptor: 모든 요청에 JWT 자동 첨부
apiClient.interceptors.request.use((config) => {
  const token = typeof window !== 'undefined' ? localStorage.getItem('accessToken') : null;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// Response Interceptor: 401 시 토큰 갱신/로그인 페이지 이동 등 처리
apiClient.interceptors.response.use(
  (res) => res,
  (err: AxiosError) => {
    if (err.response?.status === 401) {
      // 토큰 만료 시 로그인 페이지로 리다이렉트 등
    }
    return Promise.reject(err);
  }
);

export type ApiResponse<T> = { success: boolean; data?: T; message?: string; code?: string };
```

### 6.2 JWT 저장 위치

백엔드가 JWT를 발급한다면, 다음 중 하나에 저장합니다. (백엔드 개발자와 논의)

- **localStorage**: 클라이언트 전용, XSS에 취약
- **httpOnly Cookie**: 백엔드가 Set-Cookie로 설정, 보안에 유리
- **Zustand + 메모리**: 새로고침 시 초기화되므로, 토큰은 Cookie/localStorage와 함께 사용

## 7. 개발 꿀팁

| 항목 | 설명 |
|------|------|
| **Icon** | `lucide-react`는 가볍고 예쁘지만 배드민턴/셔틀콕 아이콘이 없을 수 있음. `react-icons`를 함께 설치하면 다양한 아이콘 사용 가능 |
| **Shadcn UI** | Tailwind 기반으로 버튼, 모달, 달력 등을 코드로 관리하면서 세련된 디자인을 빠르게 구현. Next.js 커뮤니티에서 널리 사용 (`npx shadcn@latest init`) |
| **Zustand** | TanStack Query는 서버 데이터용. "로그인한 유저 정보", "현재 선택한 지역" 같은 전역 UI 상태는 Zustand로 관리 (Redux보다 가벼움) |
