## Epic 9: 마이페이지·운영·계정 (프론트 대비 백엔드 보강)

> 마이페이지 등 **일부 UI는 이미 존재**할 수 있으나, **API·도메인이 없거나 부족**해 동작하지 않는 영역을 정리한다. 프론트 열은 **API 연동·화면 정합 완료** 기준. 상세 일정은 `docs/스프린트플래닝.md` Sprint 7, 백엔드 Step·작업 목록은 `docs/Sprint7-할일.md`.
>
> **구현 방향 (Sprint 7 합의)**: B9-1은 **`ProfileResponse` 집계**만. B9-2~4는 **별도 화면용 조회 API**. **B9-5~7** 은 **해당 스프린트 제외** — `docs/Sprint7-할일.md` Step 0 참고.

| ID | User Story | 우선순위 | 백엔드 | 프론트 |
|----|------------|----------|--------|--------|
| B9-1 | **활동 요약**: 사용자로서 마이페이지에서 **내가 개설한 매칭 수**와 **확정 참여한 매칭 수**(또는 정책에 따른 참여 횟수)를 볼 수 있다 | P1 | ⬜ | ⬜ |
| B9-2 | **매칭 내역(개설)**: 사용자로서 내가 방장인 매칭 목록을 상태·기간 등으로 조회할 수 있다 | P1 | ⬜ | ⬜ |
| B9-3 | **매칭 내역(참여)**: 사용자로서 내가 참여한(확정 등 정책 일치) 매칭 목록을 조회할 수 있다 | P1 | ⬜ | ⬜ |
| B9-4 | **후기 관리(마이페이지)**: 사용자로서 작성한 후기·받은 후기·작성 대기(미작성) 현황을 한곳에서 확인할 수 있다 (`docs/요구사항분석.md` 후기 공개 규칙 준수) | P1 | ⬜ | ⬜ |
| B9-5 | **공지사항**: 사용자로서 서비스 공지 목록·상세를 조회할 수 있다 (관리자 등록/정적 배포 등 구현 방식은 설계 시 확정) | P2 | ⬜ | ⬜ |
| B9-6 | **고객센터**: 사용자로서 FAQ, 문의 안내, 약관·운영 정책 링크 등을 확인할 수 있다 (1:1 문의 티켓은 선택) | P2 | ⬜ | ⬜ |
| B9-7 | **계정 관리**: 사용자로서 탈퇴(또는 비활성화) 요청, 연동 계정 정보 확인 등 계정 수명주기 API를 사용할 수 있다 | P1 | ⬜ | ⬜ |
