## 서비스 한눈에 보기

동호회·클럽 없이도 **지역·실력**을 기준으로 배드민턴 매칭을 열고 참여할 수 있게 하는 서비스다. 사용자는 **프로필·매칭 글·참여 승인·채팅·후기·패널티·제재**까지 한 흐름으로 이용하며, **마이페이지**에서 활동 지표·매칭 내역·후기·공지·계정을 관리한다.

---

## 비기능·품질 (요약)

알림·푸시 응답 시간, 개인정보·채팅 보호, 트래픽 증가 시 확장 가능한 구조 등은 스프린트·운영 단계에서 `application` 설정·인프라·API 명세와 함께 다룬다.

## 친구 및 활동 알림

### 2.1 친구 관계 모델 (단방향)

* **관계 정의**: A가 B를 친구로 추가(Follow)하더라도 B의 친구 목록에 A가 자동으로 추가되지 않음.
* **제한 사항**:
  * 자기 자신을 친구로 추가할 수 없음.
  * 중복 친구 추가 불가 (이미 추가된 대상이면 ‘이미 친구입니다’ 안내).
* **데이터 필드**: `follower_id` (나), `following_id` (친구).

### 2.2 실시간 활동 알림 규칙

친구(내가 팔로우하는 대상)의 주요 활동에 대해 실시간 알림을 발송합니다.

| 알림 트리거 이벤트 | 발송 시점 (Backend 로직) | 알림 메시지 예시 |
| :--- | :--- | :--- |
| **새 매칭 생성** | `Match` 엔티티가 `CREATED` 상태로 DB 저장이 완료된 직후 | "[친구닉네임]님이 새로운 매칭을 만들었습니다! 지금 확인해보세요." |
| **참여 확정** | `MatchParticipant`의 상태가 `WAITING` → **`CONFIRMED`**로 변경된 직후 | "[친구닉네임]님이 매칭 참여를 확정했습니다! 함께 참여하시겠어요?" |

### 2.3 알림 수신 대상자 선정 (Targeting)

* 활동을 한 유저(B)를 **친구로 등록한 모든 사용자(Followers)**에게 알림을 발송합니다.
* **쿼리 예시**: `SELECT follower_id FROM friendship WHERE following_id = :activeUserId`

### 2.4 상세 이동 로직 (Deep Link)

* 모든 친구 활동 알림에는 `relatedMatchId`를 포함합니다.
* 사용자가 알림을 클릭하면 해당 `matchId`를 가진 **매칭 상세 페이지 (`/matching/[id]`)**로 즉시 이동합니다.

## 히스토리 — 완료 요구사항 (한 줄 요약)

아래는 초기 요구사항·기획 정리를 **완료·반영된 단위**로 압축한 목록이다. 세부 표·예외·화면 항목은 백업 파일 또는 `docs/ERD.md`, `common-docs/api/`, 스프린트 할일 문서를 참고한다.

| 구분 | 한 줄 요약 |
| --- | --- |
| Phase 1~5 프레임 | 서비스 정의 → 기능·비기능 → 사용자 흐름 → 화면·데이터 명세까지 단계별 기획 가이드를 사용했다. |
| 문제·비전 | 클럽 의존·급수 불균형·급한 인원 모집 어려움을 해소하고, 누구나 자유롭게 매칭하는 문화를 목표로 했다. |
| 로그인·프로필 | 소셜·일반 로그인, 닉네임·지역·급수·라켓 등 프로필과 타인 조회, 관심 지역 최대 2곳을 요구했다. |
| 마이페이지·계정 | 개설·참여 매칭 수, 매칭 내역(개설/참여 구분), 후기 허브, 공지·고객센터, 탈퇴·계정 수명 API를 정의했다. |
| 매칭 CRUD·검색 | 생성·수정(RECRUITING·정원 하한)·목록·상세, 관심 지역·급수·시간 등 필터를 요구했다. |
| 참여·승인 | 신청 멘트, 방장 수락/거절, 알림(저장·조회 중심) 흐름을 정의했다. |
| 매칭 종료 | 방장 수동 FINISHED, 시작 시각 기준 스케줄 자동 FINISHED 등 후기·패널티 전제를 두었다. |
| 채팅·패널티 | 확정 인원 채팅, 노쇼·지각 등 방장 판단 패널티와 단계별 제재(참여 금지·정지·퇴출)를 설계했다. |
| 후기 | 상호 익명·공개 시점, 가중 평점(rating), 해시태그·3단계 평가를 요구했다. |
| Phase 2 스토리 | 사용자·매칭 코어·소통·피드백으로 카테고리화하고, Epic 9 등과 연결했다. |
| Phase 4 플로우 | 탐색 → 신청·대기열(순차·긴급) → 확정·채팅 및 거절·취소·대기 이전 등 예외를 표로 정리했다. |
| Phase 5 UI | 홈·상세·작성/수정·채팅·마이페이지 5화면과 와이어·데이터 스펙·버튼·상태 정책을 나열했다. |
| 친구·소셜 알림 | 단방향 팔로우(`follower_id`/`following_id`), 매칭 생성·참여 확정 시 팔로워에게 실시간 알림, `relatedMatchId`로 상세(`/matching/[id]`) 딥링크. |
| 기획 제안(일부) | 급수 검증, 코트비 정산 UI, 대기열 하이브리드 타임아웃, LoL식 자동 매칭은 **후속·선택**으로 남겼다. |
